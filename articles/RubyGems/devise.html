deviseを使ったユーザー管理機能

Gemfileの編集
gem 'devise'

Gemのインストール
bundle install

ローカルサーバーの再起動
rails s

設定ファイルの作成
rails g devise:install

deviseのUserモデルの作成
rails g devise user

=> routes.rbにルーティングが自動で追記（devise_for :users）

マイグレーションファイル
email, passwordの各カラムが自動生成

マイグレーションの実行
rails db:migrate

ローカルサーバーの再起動
rails s

_____________________________________
deviseのビューファイルの作成
rails g devise:vews

ヘルパーメソッド補記
<%= f.label :password %>
<%= f.password_field :password, autocomplete: "off" %>

新規登録画面：
app/views/devise/registrations/new.html.erb

ログイン画面：
app/views/devise/sessions/new.html.erb

＜メール、パスワードに加えてニックネームを登録する。＞------------

マイグレーションの生成
rails g migration AddNicknameToUsers nickname:string

反映
rails db:migrate

サーバーの再起動
rails s

________________________________________

ニックネームに6文字の文字制限を加える（新規登録画面）
<%= f.label :nickname %>(6 characters maximum)

<%= f.text_field :nickname, autofocus: true, maxlength: "6" %>

________________________________________

＜目標：ニックネームの値を受け取る＞
application_controller.rbの編集
before_action :configure_permitted_parameters, if: :devise_controller?
private
def configure_permitted_parameters
devise_parameter_sanitizer.permit(:sign_up, keys: [:nickname])
end

※ifオプション：値にメソッド名を指定することで、その戻り値がtrueであった時のみ処理を実行する。

______________________________________

＜目標：ログインの有無で処理を変える＞
user_signed_in?メソッド
ユーザーがログインしていればtrue、ログアウト状態ならばfalseを返す。

application.html.erbの編集
<% if user_signed_in? %>
<div class="user_nav grid-6">
<%= link_to "ログアウト", destroy_user_session_path, method: :delete %>
（以下略）

unless
ifと異なり、条件式がfalseの時にelseまでの処理を実行

redirect_toメソッド
redirect_to action: :リダイレクト先のアクション名

リダイレクト処理（コントローラー）
↓
before_action :move_to_index, except: [:index, :show]
（中略）
private
def move_to_index
unless user_signed_in?
redirect_to action: :index
end
end
